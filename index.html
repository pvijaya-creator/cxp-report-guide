The 8-Step Automated Process

Each week, the AI follows these steps in order — stopping at Step 5 to check with you before writing anything.

Step 1

Parse Previous Report

Reads the previous Confluence page using get_page_content. Extracts current status color, active risks, upcoming milestones, launch dates, and carry-over action items. This is the baseline everything else compares against.

Step 2

Search Data Sources — Comprehensive Approach

Queries specified Slack channels (slack_search_public_and_private) AND searches the project name broadly across ALL public channels (slack_search_public) to catch stakeholder announcements, executive decisions, and cross-team updates. Also queries the Jira epic for issues updated or completed in the last 7 days. Multiple targeted searches — never just one.

Step 3

🔎 Identify Changes

Analyzes all data found and answers: What milestones completed? What risks resolved or appeared? What decisions were made? Did launch dates change? What training or readiness items completed? Assesses whether the status color should change and builds the evidence for a confidence level recommendation.

Step 4

🤝 Recommend Status + Ask For Your Input — PAUSES HERE

Presents 3-5 key findings, recommends GREEN/YELLOW/RED with reasoning and confidence %, then explicitly asks: "Have I missed anything? Is agent training complete? Are there any new risks I didn't find?" Waits for your confirmation and corrections before writing a single word of the report.

Step 5

Update Report Sections

After your confirmation: replaces Highlights with this week's events, updates Risks table (add/remove), refreshes Milestones, moves completed items from Upcoming to Highlights, marks completed milestones as DONE, updates launch dates throughout the entire document, and adds Status History entries if color or dates changed.

Step 6

Verify CXP Compliance

Runs the full 18-point checklist before generating output: single status color, dates at end of bullets, no "we/our/I" language, no vague future tense, risks table placement, no redundancy, milestones marked DONE, future phases minimized, and more.

Step 7

Generate Output in Confluence HTML Format

Matches the EXACT formatting structure from the previous week's report — same HTML tags, table structures, CSS classes, and Confluence macros (status badges, user links, etc.). Ready to paste directly or create as a new page.

Step 8

Create New Confluence Page

Uses Nordstrom Confluence:create_page to automatically create the new page in the same space as the previous report. Title format: [DRAFT] MM/DD Project [Name] Status Email. Returns the URL for you to review and publish.

Mandatory Report Structure

Every CXP report follows this exact structure — in this exact order. Click each section to expand.

🚨 Critical: The Active Risks table MUST appear immediately after the Executive Summary — in Position 2. This is the most commonly violated CXP standard.
Section 1 — Executive Summary▼
  • Overall Status: GREEN / YELLOW / RED — one color, never "trending"
  • Confidence Level: "X% confident in [date] delivery"
  • Project Objective: exactly 1 sentence
  • Launch Timeline with key dates and traffic ramp plan
  • Status History — only if status OR launch dates changed
  • Success Metrics with specific numeric targets
⚠️ Section 2 — Active Risks Table ← CRITICAL PLACEMENT▼
  • Table columns: #, Risk, Impact, Mitigation, Owner, Due Date
  • Impact must specify both Timeline and Business impact — specifically
  • Mitigation must include concrete actions with dates
  • Only ACTIVE risks — resolved ones are removed (moved to Highlights)
  • If no active risks: state "No active risks"
Section 3 — Current Phase / Workstream — [STATUS]▼
  • 2-3 sentence summary of current state — no more
  • Highlights: last 7 days only, dates at END of bullet, chronological order
  • Format: "Event completed MM/DD" — not "MM/DD: Event"
  • Upcoming Milestones: next 2-4 weeks only, "Milestone: MM/DD" format
Section 4 — Future Phases (Appendix — Keep Minimal)▼
  • Next Phase: 2-3 sentences max
  • Subsequent Phases: 1 sentence each
  • No detailed milestones or planning content
  • Only include if multi-phase project
Section 5 — Resources & Contacts▼
  • Email Distribution group with GroupID link
  • Program Hub (Confluence link)
  • Project Tracking (Jira epic link)
  • Core Team: Product, Engineering, Program, Principal names
